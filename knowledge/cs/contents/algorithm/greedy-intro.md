## 핵심 개념

그리디(Greedy) 알고리즘은 전체를 한 번에 고려하지 않고, 지금 단계에서 가장 좋아 보이는 선택을 반복해 최종 답을 만드는 방식이다.

그리디가 올바른 답을 내려면 두 가지 성질이 필요하다.
- **탐욕 선택 속성**: 지금 최선의 선택이 전체 최적에 기여한다.
- **최적 부분 구조**: 부분 문제의 최적해가 전체 최적해에 포함된다.

입문자가 자주 틀리는 포인트는 "그리디로 풀었더니 틀렸어요"인데, 원인은 탐욕 선택 속성이 성립하지 않는 문제에 그리디를 쓴 것이다.

## 한눈에 보기

거스름돈 예시(동전: 500, 100, 50, 10원):

```
거슬러야 할 금액: 1260원
→ 500 × 2 = 1000
→ 100 × 2 = 200
→  50 × 1 =  50
→  10 × 1 =  10  (합계 1260)
동전 6개로 최소
```

| 항목 | 설명 |
|---|---|
| 선택 기준 | 큰 단위 동전부터 최대한 사용 |
| 탐욕 선택 | 남은 금액에서 현재 최대 동전 |
| 성립 조건 | 큰 단위가 작은 단위의 배수일 때 |
| 반례 | 동전이 [1, 3, 4]이고 6 만들 때: 탐욕은 4+1+1=3개, 최적은 3+3=2개 |

## 상세 분해

- **정렬이 첫 단계**: 그리디는 보통 입력을 특정 기준으로 정렬하는 것부터 시작한다. 활동 선택 문제(Activity Selection)는 종료 시간 순 정렬이 핵심이다.
- **한 번만 훑는다**: 한 방향으로 순회하면서 조건에 맞으면 선택, 아니면 건너뛴다. 되돌리거나 재고하지 않는다.
- **증명이 없으면 틀릴 수 있다**: 탐욕 선택이 맞다는 확신이 없으면 DP로 접근하는 것이 안전하다. 그리디는 빠르지만 틀리면 디버깅이 어렵다.
- **시간 복잡도**: 보통 O(n log n)이 정렬 비용이고, 선택 과정은 O(n)이다.

## 흔한 오해와 함정

- "그리디는 항상 틀릴 수 있으니 DP를 쓴다" → 탐욕 선택 속성이 성립하는 문제에서는 그리디가 훨씬 빠르다. 무조건 DP가 정답은 아니다.
- "동전 문제는 항상 그리디로 풀 수 있다" → 동전 단위가 배수 관계를 이루지 않으면 그리디가 틀린다. 임의 동전 조합 문제는 DP가 필요하다.
- 입력 크기가 크면 O(2^n) 완전 탐색은 시간 초과가 나고, 그리디나 DP를 고려해야 한다.
- 그리디가 맞는지 확인하는 실전 방법: "더 좋은 선택을 나중에 했을 때 결과가 개선되는지"를 반례로 보여주면 그리디가 틀렸다는 증거가 된다.

## 실무에서 쓰는 모습

1. **작업 스케줄링**: 마감 시간이 빠른 작업부터 처리하면(EDF, Earliest Deadline First) 마감을 넘기는 작업 수를 최소화한다.
2. **파일 압축(허프만 코딩)**: 등장 빈도가 낮은 문자에 긴 코드를, 빈도가 높은 문자에 짧은 코드를 배정할 때 항상 최소 빈도 두 개를 합치는 그리디 방식이 최적임이 증명됐다.

## 면접/시니어 질문 미리보기

- "그리디와 DP의 차이는 무엇인가요?" → 그리디는 매 단계 지역 최적을 선택하고 되돌리지 않는다. DP는 모든 경우를 저장하고 비교해 전역 최적을 찾는다. 그리디는 빠르지만 적용 가능 조건이 제한적이다.
- "탐욕 선택 속성이 성립하는지 어떻게 확인하나요?" → exchange argument를 쓴다. "내 선택보다 나은 선택이 있다면, 두 선택을 교환해도 결과가 나빠지지 않는다"를 보이면 탐욕 선택이 안전하다는 증거가 된다.
- "그리디를 쓸 수 있는 문제의 특징은?" → 선택의 결과가 이후 선택에 독립적이거나, 가장 좋은 것을 먼저 처리하면 다른 것도 자동으로 개선되는 구조일 때다.

## 한 줄 정리

그리디는 매 단계 지금 최선을 선택하는 방법이며, 탐욕 선택 속성이 성립한다는 확신이 있을 때만 DP 대신 쓸 수 있다.
