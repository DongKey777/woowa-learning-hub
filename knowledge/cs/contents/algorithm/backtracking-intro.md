## 핵심 개념

백트래킹(Backtracking)은 완전 탐색의 개선판이다. 재귀로 탐색하되, 현재까지의 선택이 조건을 만족할 수 없다고 판단되면 그 즉시 되돌아가(backtrack) 다른 경로를 탐색한다.

핵심 아이디어는 **가지치기(pruning)** 다. "지금 이 방향으로 계속 가면 절대 정답이 나오지 않는다"고 판단하는 순간 탐색을 중단한다. 이 판단이 빠르고 정확할수록 백트래킹이 강력해진다.

입문자가 헷갈리는 점은 완전 탐색과 백트래킹의 경계다. 완전 탐색은 모든 경우를 끝까지 가고, 백트래킹은 틀린 경로를 중간에 끊는다. 실제 구현은 재귀 함수 안에 조건 검사를 추가하는 것뿐이다.

## 한눈에 보기

N-Queen(4×4): 1행에 퀸을 놓고 충돌 없는 열을 재귀로 탐색

```
1행 1열 배치 → 2행 탐색
  2행 1열: 충돌! 가지치기
  2행 2열: 충돌! 가지치기
  2행 3열: OK → 3행 탐색
    3행 1열: OK → 4행 탐색
      4행: 모두 충돌 → 되돌아가기
    ...
  2행 4열: ...
```

| 항목 | 완전 탐색 | 백트래킹 |
|---|---|---|
| 탐색 범위 | 모든 경우 | 유망한 경로만 |
| 종료 시점 | 항상 끝까지 | 불가 판단 즉시 중단 |
| 구현 추가 사항 | 없음 | 가지치기 조건 |
| 시간 복잡도 | 이론 worst | 실제로 훨씬 빠름 |

## 상세 분해

- **재귀 + visited 배열**: 현재 선택한 원소를 표시하고 재귀 호출 후 표시를 해제한다. 해제(unmark)를 빠뜨리면 다른 경로에서 잘못된 visited 상태를 물려받는다.
- **가지치기 조건**: 재귀 호출 전에 "현재 상태가 유효한가"를 검사한다. 무효면 바로 return한다. N-Queen의 경우 같은 열, 대각선에 퀸이 있으면 무효다.
- **Base case 정의**: 모든 자리를 채웠거나(성공), 더 이상 진행 불가(실패)일 때 재귀를 멈춘다.
- **원상 복구(undo)**: 선택을 취소할 때는 선택 전 상태를 정확히 복구해야 한다. visited 해제, 카운터 감소, 배열 원소 제거 등.

## 흔한 오해와 함정

- "백트래킹은 느리다" → 가지치기가 충분하면 완전 탐색보다 훨씬 빠르다. 실제 N-Queen의 경우 pruning으로 탐색 노드가 수십 배 줄어든다.
- "visited 해제를 안 해도 된다" → 재귀를 통해 다른 경로로 넘어갈 때 이전 경로의 visited 표시가 남아 있으면 탐색 범위가 잘못 좁혀진다. 반드시 해제해야 한다.
- 가지치기 조건을 재귀 호출 후에 검사하면 의미가 없다. 호출 전에 검사해야 불필요한 호출을 막는다.
- 재귀 깊이가 입력 크기에 비례하므로 n이 1000 이상이면 스택 오버플로우를 고려해야 한다.

## 실무에서 쓰는 모습

1. **순열/조합 생성**: 중복 없이 n개 중 k개를 고르는 조합을 재귀로 구현할 때 이미 고른 인덱스 이전으로 돌아가지 않는 가지치기로 경우의 수를 절반 이하로 줄인다.
2. **스도쿠 풀이**: 빈 칸에 1~9를 넣어보고 행·열·박스 중복이 생기면 즉시 돌아간다. 가지치기가 없으면 9^81이지만 실제 탐색은 매우 빠르게 완료된다.

## 면접/시니어 질문 미리보기

- "백트래킹과 DFS의 차이는?" → DFS는 그래프 탐색 알고리즘이고, 백트래킹은 탐색 전략이다. 백트래킹은 DFS 방식으로 탐색하되 유망하지 않으면 즉시 되돌아간다.
- "백트래킹이 완전 탐색보다 항상 빠른가요?" → 가지치기 조건이 없거나 모든 경로가 유망하면 성능 차이가 없다. 가지치기 조건의 품질이 곧 백트래킹의 효율이다.
- "N-Queen의 시간 복잡도는?" → 이론적으로 O(n!)이지만 가지치기로 실제 탐색 노드는 n=12일 때 전체 12!의 0.1% 미만이다.

## 한 줄 정리

백트래킹은 완전 탐색 + 가지치기이며, "지금 이 경로는 틀렸다"고 판단하는 순간 되돌아가 불필요한 탐색을 건너뛰는 방법이다.
