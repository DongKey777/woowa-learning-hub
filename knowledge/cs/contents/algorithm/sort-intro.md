## 핵심 개념

정렬은 데이터를 특정 기준(오름차순, 내림차순 등)으로 재배열하는 연산이다. 어떤 정렬 알고리즘을 고를지는 두 가지로 결정된다.

- **시간복잡도**: 비교 횟수가 n², n log n 중 어느 쪽인가.
- **안정성(stability)**: 동일 키를 가진 원소의 원래 순서를 보존하는가.

입문자가 자주 놓치는 점은 "안정 정렬이 필요한 경우"다. 이름으로 정렬한 뒤 나이로 한 번 더 정렬할 때, 나이가 같은 사람끼리 이름 순서가 유지되어야 한다면 안정 정렬이 필요하다.

## 한눈에 보기

| 알고리즘 | 평균 복잡도 | 최악 복잡도 | 안정 여부 | 제자리 여부 |
|---|---|---|---|---|
| Bubble Sort | O(n²) | O(n²) | 안정 | O(1) 추가 공간 |
| Selection Sort | O(n²) | O(n²) | 불안정 | O(1) 추가 공간 |
| Insertion Sort | O(n²) | O(n²) | 안정 | O(1) 추가 공간 |
| Merge Sort | O(n log n) | O(n log n) | 안정 | O(n) 추가 공간 |
| Quick Sort | O(n log n) | O(n²) | 불안정 | O(log n) 스택 |

Java의 `Arrays.sort`는 기본 타입에 QuickSort 계열, 객체에 TimSort(안정 정렬)를 사용한다.

## 처음 붙일 안전한 큰 그림

정렬을 처음 배우면 "`버블 정렬부터 다 외워야 하나요?`"에서 자주 멈춘다. 처음에는 알고리즘 이름을 다 외우기보다 **"왜 정렬하나 -> 어떤 기준으로 고르나 -> 정렬 뒤 무엇을 하나"**만 잡는 편이 안전하다.

| 지금 드는 질문 | 먼저 붙일 답 |
|---|---|
| 정렬이 왜 필요해요? | 검색, 순위, 중복 처리 전에 순서를 맞추는 전처리다. |
| 어떤 정렬을 써야 해요? | 코딩 테스트와 실무에서는 먼저 언어 내장 정렬을 쓰고, 안정성/추가 메모리만 확인한다. |
| 정렬 다음엔 뭘 보나요? | 정렬된 결과를 다시 찾거나 경계를 고를 때 [이분 탐색 패턴](./binary-search-patterns.md)으로 이어진다. |
| Java 코드에서 바로 써보고 싶어요 | `Arrays.sort(...)`와 검색 흐름은 [Sorting and Searching Arrays Basics](../language/java/java-array-sorting-searching-basics.md)부터 붙인다. |

## 상세 분해

- **버블 정렬**: 인접 원소를 반복 교환. 가장 이해하기 쉽지만 가장 느리다. O(n²) 비교.
- **선택 정렬**: 매 단계에서 최솟값을 찾아 앞으로 옮긴다. 항상 O(n²)이고 교환 횟수가 적다.
- **삽입 정렬**: 현재 원소를 이미 정렬된 앞쪽 부분에 삽입. 거의 정렬된 배열에서 O(n)에 가깝다.
- **병합 정렬**: 반으로 나눠 각각 정렬 후 합친다. 항상 O(n log n)이고 안정적이지만 추가 메모리가 필요하다.
- **퀵 정렬**: 피벗을 기준으로 작은/큰 쪽으로 분할. 평균 O(n log n)이지만 피벗 선택이 나쁘면 최악 O(n²).

## 흔한 오해와 함정

- "정렬은 버블/선택/삽입부터 다 구현할 줄 알아야 한다" → 처음에는 아니다. 먼저 "내장 정렬을 써도 되는지", "안정 정렬이 필요한지"를 판단하는 쪽이 더 중요하다.
- "퀵 정렬이 항상 빠르다" → 이미 정렬된 배열에서 피벗을 항상 최솟값으로 고르면 O(n²)이 된다. 랜덤 피벗이나 median-of-three로 완화한다.
- "병합 정렬이 느리다" → 최악도 O(n log n)으로 안정적이다. 다만 추가 O(n) 공간이 필요하다.
- "삽입 정렬은 쓸모없다" → 거의 정렬된 소규모 데이터에서는 오버헤드가 적어 TimSort가 내부적으로 사용한다.
- 정렬 후 `Arrays.sort`의 결과 순서를 정렬 알고리즘 이름으로 설명하면 틀릴 수 있다. 언어마다 내부 구현이 다르다.

## 실무에서 쓰는 모습

1. **객체 리스트 정렬**: Java에서 `Comparator`를 전달해 `Collections.sort`를 호출. 내부적으로 TimSort(안정 정렬)가 사용된다.
2. **특수 키 정렬**: 등록 시간이 같으면 이름 순으로 정렬해야 할 때 안정 정렬 + 복합 `Comparator`를 쓴다.

## 면접/시니어 질문 미리보기

- "비교 기반 정렬의 하한이 O(n log n)인 이유는?" → 모든 정렬 알고리즘을 결정 트리로 표현하면 리프 노드가 n!개 이상이어야 하고, 트리 높이가 최소 log₂(n!) ≈ n log n이다.
- "QuickSort가 실제로 MergeSort보다 빠른 이유는?" → 평균 복잡도가 같아도 캐시 지역성이 좋고 추가 메모리가 없어서 상수 계수가 작다.
- "안정 정렬이 왜 중요한가요?" → 다중 기준 정렬을 여러 번 적용할 때 이전 기준 순서가 보존되어야 올바른 결과가 나온다.

## 한 줄 정리

정렬 알고리즘은 시간복잡도(O(n²) vs O(n log n))와 안정성으로 구분하며, 실무에서는 언어 내장 정렬을 쓰되 안정 여부와 피벗 선택 트레이드오프를 설명할 수 있어야 한다.
