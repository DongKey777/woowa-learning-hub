## 핵심 개념

트리는 **노드(node)와 간선(edge)으로 이루어진 계층 구조**다.
다른 그래프와 달리 트리는 **사이클이 없고**, 루트에서 임의의 노드까지 정확히 하나의 경로만 존재한다.

입문자가 자주 헷갈리는 포인트는 이것이다.

- 그래프는 사이클이 있을 수 있지만, 트리는 사이클이 없다.
- 연결 리스트는 트리의 특수한 형태(각 노드의 자식이 최대 1개)로 볼 수 있다.

핵심 용어를 먼저 잡아두면 이후 설명이 쉽다.

- **루트(Root)**: 트리의 최상단 노드. 부모가 없다.
- **리프(Leaf)**: 자식이 없는 노드.
- **깊이(Depth)**: 루트에서 특정 노드까지의 간선 수.
- **높이(Height)**: 루트에서 가장 먼 리프까지의 간선 수.

## 한눈에 보기

```text
        A          ← 루트 (깊이 0)
       / \
      B   C        ← 깊이 1
     / \
    D   E          ← 깊이 2, D와 E는 리프
```

| 용어 | 설명 |
|---|---|
| 루트 | A (부모 없음) |
| 리프 | D, E, C (자식 없음) |
| B의 부모 | A |
| B의 자식 | D, E |
| 트리 높이 | 2 |

## 상세 분해

트리에서 자주 나오는 세 가지 개념을 나눠서 보자.

- **이진 트리(Binary Tree)**: 각 노드가 최대 2개의 자식을 가진다. 가장 기본적인 트리 형태다.
- **이진 탐색 트리(BST)**: 이진 트리 + "왼쪽 자식 < 부모 < 오른쪽 자식" 규칙이 추가된다. 탐색이 O(log n)으로 빨라진다.
- **완전 이진 트리(Complete Binary Tree)**: 마지막 레벨을 제외하고 모든 레벨이 꽉 찬 트리. 힙(Heap)이 이 구조를 쓴다.

## 흔한 오해와 함정

- **오해 1: 트리는 항상 균형 잡혀 있다.**
  균형 여부는 별개 문제다. 일반 이진 트리는 편향(skewed)될 수 있고, 그러면 탐색이 O(n)으로 느려진다.

- **오해 2: 깊이와 높이가 같은 개념이다.**
  깊이는 루트 기준 내려간 거리, 높이는 리프 기준 올라간 거리다. 같은 트리에서 노드마다 깊이가 다르다.

- **함정: 리프 조건을 놓친다.**
  트리 순회 중 리프 판단은 `node.left == null && node.right == null`이다. 둘 중 하나만 null인 노드는 리프가 아니다.

## 실무에서 쓰는 모습

**파일 시스템** — 디렉토리 구조가 트리다. 루트(`/`)에서 출발해 하위 디렉토리로 내려가는 계층이 트리의 부모-자식 관계 그대로다.

**HTML DOM** — 웹 페이지의 `<html>` 태그가 루트이고, `<head>`, `<body>` 등이 자식 노드다. 브라우저가 DOM 트리를 순회하며 렌더링한다.

## 면접/시니어 질문 미리보기

1. **트리와 그래프의 차이는 무엇인가요?**
   트리는 사이클이 없고 루트가 하나인 연결 그래프다. 그래프는 사이클이 있을 수 있고 루트 개념이 없을 수 있다. 트리는 그래프의 특수한 형태다.

2. **이진 트리와 이진 탐색 트리의 차이는?**
   이진 트리는 자식이 최대 2개인 구조다. BST는 이진 트리에 "왼쪽 < 부모 < 오른쪽" 정렬 규칙이 추가된 것으로, 탐색·삽입·삭제가 O(log n)이 된다.

3. **트리의 높이가 왜 중요한가요?**
   탐색 시간복잡도가 높이에 비례하기 때문이다. 균형 트리는 높이가 O(log n)이지만, 편향 트리는 O(n)까지 나빠질 수 있다.

## 한 줄 정리

트리는 사이클 없는 계층 구조이며, 루트·깊이·높이·리프 4개 용어를 이해하면 이진 트리·BST·힙까지 자연스럽게 확장할 수 있다.
