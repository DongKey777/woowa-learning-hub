## 핵심 개념

OSI(Open Systems Interconnection) 모델은 네트워크 통신을 7개 계층으로 나눠서 설명하는 참조 모델이다. 입문자가 가장 헷갈리는 점은 "OSI 7계층이 실제 구현이냐"다. 아니다. 실제 인터넷은 주로 TCP/IP 4계층 모델로 구현된다. OSI 7계층은 **개념 정리와 장애 분류**를 위한 공통 언어다.

## 한눈에 보기

7계층은 위(응용)에서 아래(물리)로 내려갈수록 하드웨어에 가까워진다.

```
7. 응용 계층   (Application)   HTTP, SMTP, DNS 같은 애플리케이션 프로토콜
6. 표현 계층   (Presentation)  인코딩, 암호화/복호화, 압축
5. 세션 계층   (Session)       연결 유지/복구, 대화 제어
4. 전송 계층   (Transport)     TCP / UDP, 포트 번호, 신뢰성
3. 네트워크 계층 (Network)     IP 주소, 라우팅
2. 데이터링크  (Data Link)     MAC 주소, 이더넷 프레임, 스위치
1. 물리 계층   (Physical)      전기 신호, 광 케이블, 무선
```

숫자가 **낮을수록 하드웨어에 가깝고**, 높을수록 애플리케이션에 가깝다.

## 상세 분해

### 백엔드 개발자가 자주 만나는 계층

- **7계층(응용)**: HTTP, HTTPS, WebSocket, gRPC 등. 서버 코드가 직접 다루는 영역이다.
- **4계층(전송)**: TCP/UDP, 포트 번호. 서버 소켓을 열 때 포트를 지정한다.
- **3계층(네트워크)**: IP 주소. 서버 IP를 설정하거나 방화벽 규칙을 볼 때 등장한다.

### 계층 간 관계

데이터를 보낼 때 위 계층에서 아래로 내려가며 각 계층이 헤더를 붙인다(캡슐화). 받을 때는 반대로 아래에서 위로 올라가며 헤더를 벗긴다(역캡슐화).

예: HTTP 요청이 전송되면
- 7계층이 HTTP 메시지를 만든다.
- 4계층이 TCP 세그먼트로 감싼다.
- 3계층이 IP 패킷으로 감싼다.
- 2계층이 이더넷 프레임으로 감싼다.
- 1계층이 전기/광 신호로 내보낸다.

### TCP/IP 4계층과의 관계

실무에서 자주 보는 TCP/IP 모델은 OSI 7계층을 4개로 합쳐서 이해한다.

- 응용(7+6+5) → TCP/IP 응용 계층
- 전송(4) → TCP/IP 전송 계층
- 인터넷(3) → TCP/IP 인터넷 계층
- 네트워크 접근(2+1) → TCP/IP 네트워크 접근 계층

## 흔한 오해와 함정

- "OSI 7계층은 실제로 구현된 것이다"라고 생각하면 안 된다. 실제 프로토콜 스택은 TCP/IP다.
- 계층 번호를 외우는 것보다 **어느 계층에 어떤 책임이 있는지** 구분하는 게 훨씬 중요하다.
- 계층이 독립적이라는 말은 4계층을 바꿔도 7계층 코드는 그대로여야 한다는 의미다. 예를 들어 HTTP는 TCP 위에서 동작하지만, 코드 레벨에서 HTTP는 TCP 소켓의 구현을 모른다.

## 실무에서 쓰는 모습

- 네트워크 장애가 생겼을 때 "L4(4계층) 문제인가, L7(7계층) 문제인가"를 구분하는 언어로 OSI를 쓴다.
- AWS 로드밸런서 선택 시 "L4 로드밸런서(NLB)냐 L7 로드밸런서(ALB)냐"처럼 계층 번호를 직접 쓴다.
- 방화벽 규칙을 쓸 때도 "3계층(IP) 기준인가, 4계층(TCP 포트) 기준인가"로 구분한다.

## 면접/시니어 질문 미리보기

**Q. OSI 7계층을 왜 알아야 하나요?**
계층 모델을 알면 "IP가 안 된다", "포트가 막혔다", "HTTP가 느리다" 같은 증상을 어느 계층 문제인지로 분리해 원인을 좁힐 수 있다.

**Q. HTTP는 몇 계층 프로토콜인가요?**
7계층(응용 계층)이다. 내부적으로 4계층 TCP(또는 QUIC/UDP), 3계층 IP를 사용하지만 HTTP 자체의 정의는 응용 계층이다.

**Q. L4와 L7 로드밸런서 차이는 무엇인가요?**
L4는 IP/포트 기준으로 부하를 분산하고, L7은 HTTP 헤더나 경로 등 응용 계층 정보까지 보고 분산한다.

## 한 줄 정리

OSI 7계층은 실제 구현이 아닌 참조 모델이며, 계층별 책임 구분을 알면 장애 원인을 좁히는 공통 언어가 된다.
