# 애플리케이션 탭에는 쿠키가 보이는데 요청 헤더는 비어 있음 원인 라우터

## 한 줄 요약

> `Application > Cookies`는 저장 사실이고 request `Cookie` header는 이번 전송 판정이다. 쿠키가 보이는데 요청에 안 붙으면 "없다"보다 "전송 조건이 안 맞는다" 쪽에서 시작해야 한다.

## 가능한 원인

1. cross-origin `fetch`나 axios 호출에서 credential 전송 옵션이 빠졌을 수 있다. 이 경우 브라우저는 저장된 cookie를 가지고 있어도 이번 요청에 자동으로 싣지 않는다. 다음 문서: [Fetch Credentials vs Cookie Scope](../security/fetch-credentials-vs-cookie-scope.md), [Cross-Origin Cookie, `fetch credentials`, CORS 입문](./cross-origin-cookie-credentials-cors-primer.md)
2. cookie `Domain`, `Path`, `SameSite` 범위가 현재 요청 URL과 안 맞을 수 있다. 특히 `auth.example.com`에 저장된 host-only cookie를 `app.example.com`이나 다른 path에서 기대하면 Application 탭에는 보여도 request header는 비어 있을 수 있다. 다음 문서: [Cookie Scope Mismatch Guide](../security/cookie-scope-mismatch-guide.md), [Application 탭에는 Cookie가 보이는데 Request `Cookie` 헤더는 비는 이유 미니 카드](./application-tab-vs-request-cookie-header-mini-card.md)
3. `Secure` cookie인데 실제 요청 URL이 `http://`로 꺾이거나 프록시 뒤에서 앱이 잘못된 scheme를 보고 있을 수 있다. 이때 저장 row는 남지만 HTTPS가 아닌 실제 요청에는 cookie가 안 붙는다. 다음 문서: [Secure Cookie Behind Proxy Guide](../security/secure-cookie-behind-proxy-guide.md)
4. 실제로는 cookie가 빠진 뒤 login redirect나 hidden HTML 응답으로 이어졌는데, 학습자가 마지막 응답만 보고 "쿠키가 사라졌다"고 오해했을 수 있다. request `Cookie`가 빈 순간과 `/login` 또는 HTML fallback을 같이 봐야 한다. 다음 문서: [Login Redirect, Hidden `JSESSIONID`, `SavedRequest` 입문](./login-redirect-hidden-jsessionid-savedrequest-primer.md), [JSON 기대했는데 HTML 응답이 옴 원인 라우터](./json-expected-but-html-response-symptom-router.md)

## 빠른 자기 진단

1. 같은 실패 요청 row에서 `Request Headers > Cookie`가 정말 비었는지 먼저 고정한다. 다른 요청 row를 보면 과거 성공 요청과 섞여 오진하기 쉽다.
2. 요청이 cross-origin이면 코드에서 `credentials: "include"` 또는 동등한 설정이 있는지 본다. 이 옵션이 없으면 scope를 오래 봐도 답이 안 나온다.
3. Application 탭의 cookie `Domain`, `Path`, `SameSite`, `Secure`를 요청 URL과 한 줄로 비교한다. 저장 host와 요청 host, 저장 path와 요청 path가 다르면 저장 성공과 전송 성공은 분리된다.
4. 다음 요청 URL이 `https://`인지 확인한다. 특히 로드밸런서나 reverse proxy 뒤에서만 깨지면 `Secure` cookie와 scheme drift를 먼저 의심하는 편이 빠르다.
5. request `Cookie`가 빈 뒤 최종 응답이 `/login`이나 HTML인지 본다. 그러면 "서버 세션이 사라졌다"보다 "브라우저가 credential을 못 실었다"는 갈래가 더 앞선다.
