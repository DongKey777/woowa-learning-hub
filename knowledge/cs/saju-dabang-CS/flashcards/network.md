# Network Flashcards

- Q: 사주다방이 SSE를 쓴 이유는?
  A: 서버가 운세 생성 중간 결과를 단방향으로 계속 푸시해야 했기 때문

- Q: WebSocket 대신 SSE를 택한 이유는?
  A: 양방향 상호작용보다 서버→클라이언트 스트림이 핵심이었고 구현이 더 단순했기 때문

- Q: replay가 필요한 이유는?
  A: 늦게 붙거나 끊겼다가 복구된 클라이언트에 이벤트 일부를 복원하기 위해

- Q: 도메인의 역할은?
  A: 사람이 읽는 이름, DNS가 실제 서버 IP에 매핑

- Q: HTTPS와 mTLS 차이는?
  A: HTTPS는 서버 인증, mTLS는 서버와 클라이언트 모두 인증
