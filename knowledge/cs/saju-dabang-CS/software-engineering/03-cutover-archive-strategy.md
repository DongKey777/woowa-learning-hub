# 컷오버와 아카이브 전략

## 왜 바로 삭제하지 않았나

기존 Node 백엔드를 바로 지우면:
- 비교가 어려움
- 롤백 어려움
- 문서/스크립트 경로가 깨질 수 있음

그래서:
- `archive/legacy-node-api/`
- `archive/legacy-node-docs/`
로 이동
- `api/`는 심볼릭 링크 유지

## 이게 소프트웨어 공학인 이유

마이그레이션은 코드만 바꾸는 게 아니라:
- 운영 전환
- 문서 기준선 교체
- 롤백 전략

까지 포함해야 하기 때문

## 면접 포인트

“좋은 전환 전략은 현재 기준선을 명확히 하면서도, 레거시를 즉시 파괴하지 않는 방식이다.”
